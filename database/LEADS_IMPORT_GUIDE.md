# Why lead fields show blank & how to fix it

## Why Person Name, City, State (and others) show "—"

1. **Where the app gets the data**  
   The dashboard and lead detail page read:
   - **Person Name** from `lead_stage_data` → stage `lead_details` → `data.person_name`
   - **City** and **State** from `lead_stage_data` → stage `contacted` → `data.city`, `data.state`

2. **Why they’re blank**
   - **Add Lead (UI)** only creates a row in `leads` (company, stage, POC). It does **not** create or fill `lead_stage_data`. So there is no `person_name`, `city`, `state` until someone edits the “Lead” / “Contacted” stages and saves.
   - **Imports** that only insert into `leads` (or that insert empty `{}` into `lead_stage_data`) also leave Person Name, City, State blank.
   - **Saving one stage** used to replace the whole stage `data`, so other fields could be wiped. That’s fixed: the backend now **merges** new data into existing stage data so partial saves don’t clear other fields.

So blanks are almost always because:
- The lead was created without any stage data, or  
- The import didn’t map your sheet columns into `lead_details` and `contacted`.

## How to fix existing data

### Option A: Re-import using the TSV script (recommended for bulk data)

1. **Export your sheet as TSV**  
   Save as “Tab-separated values” or copy rows and paste into a `.tsv` file. Column order should be:
   - Col 0: Company Name  
   - Col 1: Status (Open/Closed)  
   - Col 2: Stage  
   - Col 3: POC (Assigned POC full name)  
   - Col 4: Person Name  
   - Col 5: Mobile  
   - Col 6: Email  
   - Col 7: Lead Generation Date (DD/MM/YYYY)  
   - Col 8: Lead Generated By  
   - Col 9: Contacted Date (optional)  
   - Col 10: Relevant  
   - Col 11: City  
   - Col 12: State  
   - Col 13–18: Industry, No of Divisions, Name of Divisions, Designation, Company Description, Inventory type  
   - … later columns for brochure, demo schedule, demo completed, etc.  
   - **Last column:** Remark (follow-up notes)

2. **Adjust column mapping if needed**  
   Open `database/tsv_to_lead_import_sql.py` and edit the `COLS` dict if your export has a different column order.

3. **Generate SQL**
   ```bash
   cd "c:\Support FMS to APPLICATION"
   python database/tsv_to_lead_import_sql.py your_leads.tsv -o database/IMPORT_LEADS_NEW.sql
   ```
   Or paste TSV into a file and run:
   ```bash
   python database/tsv_to_lead_import_sql.py < pasted_leads.txt -o database/IMPORT_LEADS_NEW.sql
   ```

4. **Run the SQL in Supabase**  
   - Open Supabase → SQL Editor.  
   - Paste the contents of `IMPORT_LEADS_NEW.sql`.  
   - Execute.  
   This creates **new** leads with reference numbers like IMP0001, IMP0002, and fills `lead_details` and `contacted` (and other stages) so Person Name, City, State show correctly.

5. **If you already have leads and want to update instead of duplicate**  
   The script currently only **inserts** new rows. To “fix” existing leads you can either:
   - Use the script to generate SQL for new leads only (e.g. after deleting wrong ones), or  
   - Manually edit a lead in the app: open the lead → Edit “Lead” / “Contacted” → fill Person Name, City, State, etc. → Save. The merge fix ensures other fields are not cleared.

### Option B: Fill in the app

For a few leads: open each lead → click **Edit** on “Lead” and “Contacted” → enter Person Name, City, State, and any other fields → Save. Data is stored in `lead_stage_data` and will show on the dashboard and detail page.

## Summary

| Cause | Fix |
|-------|-----|
| Lead created via “Add Lead” only | Edit the “Lead” and “Contacted” stages and save, or re-import with the TSV script. |
| Import didn’t map columns to lead_details/contacted | Re-import using `tsv_to_lead_import_sql.py` with the correct column order (edit `COLS` if needed). |
| Saving one stage used to wipe others | Already fixed: backend merges payload into existing stage data. |

After a correct import or after filling and saving the stages, Person Name, City, and State will show with proper IDs (lead id + stage data) and no more blank “—” for those fields.
